### Cypher 语句解释

这条语句用于查询“关联电影数量大于 2 的人物”，并按关联电影数量从高到低排序，最多返回前 10 条。

逐句解释如下：

```cypher
MATCH (p:person)-[r]->(m)
```

匹配所有从 `person` 节点 `p` 出发，通过任意关系 `r` 指向节点 `m` 的路径。

```cypher
WITH p, COUNT(m) AS movieCount
```

以人物节点 `p` 为分组对象，统计每个人关联到的节点数量，并命名为 `movieCount`。

```cypher
WHERE movieCount > 2
```

只保留关联数量大于 2 的人物。

```cypher
RETURN p.name, movieCount
```

返回人物姓名和对应的关联数量。

```cypher
ORDER BY movieCount DESC
LIMIT 10
```

按照关联数量降序排列，并只显示前 10 个结果。

### 本步骤需要用到的代码片段

#### 1. 解析输入参数

```python

```

作用：从 TuGraph 调用插件时传入的 JSON 字符串中读取 Node2Vec 所需参数。

#### 2. 创建只读事务

```python
txn = db.CreateReadTxn()
```

作用：创建只读事务，用于安全读取图中的节点和边。

#### 3. 获取邻居节点

```python
neighbor_cache = {}

def get_neighbors(vid):
    if vid not in neighbor_cache:
        v = txn.GetVertexIterator(vid)
        nbrs = []
        if v.IsValid():
            edge_it = v.GetOutEdgeIterator()
            while edge_it.IsValid():
                nbrs.append(edge_it.GetDst())
                edge_it.Next()
        neighbor_cache[vid] = nbrs
    return neighbor_cache[vid]
```

作用：

- 根据节点 ID 获取该节点的出边邻居。
- 使用 `neighbor_cache` 缓存邻居，避免重复访问数据库，提高采样效率。

#### 4. Node2Vec 有偏随机游走

```python
def biased_walk(start_node, length):
    walk = [start_node]
    curr = start_node
    prev = start_node

    for _ in range(length - 1):
        nbrs = get_neighbors(curr)
        if not nbrs:
            break

        prev_nbrs_set = set(get_neighbors(prev))

        weights = []
        for n in nbrs:
            if n == prev:
                weights.append(1.0 / p)
            elif n in prev_nbrs_set:
                weights.append(1.0)
            else:
                weights.append(1.0 / q)

        total_w = sum(weights)
        norm_weights = [w / total_w for w in weights]

        next_node = random.choices(nbrs, weights=norm_weights, k=1)[0]

        walk.append(next_node)
        prev = curr
        curr = next_node
    return walk
```

作用：实现 Node2Vec 的二阶随机游走逻辑。

其中：

- 如果候选节点 `n` 等于上一个节点 `prev`，权重为 `1 / p`，表示返回上一节点。
- 如果候选节点 `n` 是上一节点的邻居，权重为 `1`，偏向局部 BFS 搜索。
- 如果候选节点 `n` 不是上一节点的邻居，权重为 `1 / q`，偏向远距离 DFS 搜索。

#### 5. 多次采样并返回结果

```python
all_walks = []
for _ in range(num_walks):
    all_walks.append(biased_walk(start_vid, walk_len))

txn.Abort()

return (True, str(all_walks))
```

作用：

- 从同一个起点重复执行多次随机游走。
- 将所有采样路径返回给 TuGraph。
- 释放只读事务。

## 四、第 3 步：完整实现 Node2Vec 并得到嵌入向量

### 目标

Node2Vec 完整流程包括两部分：

1. 随机游走采样。
2. 将采样序列输入 Word2Vec/Skip-Gram 模型，训练得到节点嵌入向量。

当前 `ceshi_node2vec.py` 已经实现第 1 部分：随机游走采样。

要完整满足作业要求，需要在采样结果基础上继续训练嵌入向量。

### 完整 Node2Vec 流程

```text
图数据导入 TuGraph
        ↓
从 TuGraph 读取邻居
        ↓
执行 Node2Vec 有偏随机游走
        ↓
得到节点序列 walks
        ↓
使用 Word2Vec 训练节点向量
        ↓
输出每个节点的 embedding
```

现象：如果直接传入普通文本，`json.loads(input)` 会解析失败。

```json
{"start_vid": 100, "walk_len": 10, "num_walks": 5, "p": 1.0, "q": 1.0}
```

#### 问题 3：某些节点没有出边，随机游走提前结束

相关代码：

```python
nbrs = get_neighbors(curr)
if not nbrs:
    break
```

原因：如果当前节点没有邻居，就无法继续游走。

解决方法：

- 检查导入的边数据是否完整。
- 换一个有出边的 `start_vid`。
- 如果需要无向效果，可以导入反向边。

#### 问题 5：节点 ID 需要转换为字符串

原因：Word2Vec 的输入通常是 token 序列，节点 ID 如果是整数，可能不符合预期。

解决方法：

## 五、可选补充：图算法代码说明

虽然作业重点是 Node2Vec，但报告中可以补充说明本目录还包含其他图算法代码，用于展示对“图查询与图算法”主题的理解。

### 1. PageRank：`pagerank_4.py`

用途：计算图中最重要的节点。

核心代码片段：

```python
self.d = 0.85
self.one_over_all = 1.0 / num_vertices
```

含义：设置 PageRank 阻尼系数为 `0.85`，并初始化每个节点的基础权重。

核心迭代片段：

```python
self.dangling = g.ProcessVertexInRange[cython.double, PageRankCore](self.Dangling, 0, num_vertices, self)
self.dangling /= num_vertices
delta = g.ProcessVertexInRange[cython.double, PageRankCore](self.Work, 0, num_vertices, self)
self.pr_next.Swap(self.pr_curr)
```

含义：每轮迭代处理悬挂节点、更新 PageRank 值，并交换当前向量和下一轮向量。

返回结果片段：

```python
response["max_pr_vi"] = max_pr_vi
response["max_pr"] = max_pr
response["num_vertices"] = olapondb.NumVertices()
response["num_edges"] = olapondb.NumEdges()
```

含义：返回 PageRank 最大的节点及其分数。

### 2. LPA 标签传播：`lpa_2.py`

用途：进行社区检测。

初始化标签片段：

```python
for vi in range(num_vertices):
    self.label_curr[vi] = vi
```

含义：开始时每个节点的标签等于自己的节点编号。

标签更新片段：

```python
if v > max_n or (v == max_n and k < max_l):
    max_n = v
    max_l = k
self.label_next[vi] = max_l
```

含义：选择邻居中出现次数最多的标签作为当前节点的新标签；如果次数相同，选择编号较小的标签。

返回结果片段：

```python
response["max_community"] = max_community
response["num_communities"] = num_communities
response["modularity"] = modularity
response["num_vertices"] = olapondb.NumVertices()
response["num_edges"] = olapondb.NumEdges()
```

含义：返回社区数量、最大社区规模、节点数和边数。

### 3. LCC 局部聚类系数：`lcc_2.py`

用途：计算图中节点邻居之间的紧密程度。

三角形计数片段：

```python
write_add(num_triangle.begin() + a, 1)
write_add(num_triangle.begin() + b, 1)
write_add(num_triangle.begin() + pre_a, 1)
local_count += 1
```

含义：当两个邻接表中发现共同邻居时，说明形成一个三角形，对相关节点的三角形数量加一。

计算 LCC 分数片段：

```python
self.score[vi] = 2.0 * self.num_triangle[vi] / valid_degree / (valid_degree - 1)
```

含义：根据公式计算节点的局部聚类系数。

返回结果片段：

```python
response["average_lcc"] = average_lcc
response["num_vertices"] = olapondb.NumVertices()
response["num_edges"] = olapondb.NumEdges()
```

含义：返回全图平均局部聚类系数、节点数和边数。

1. 先用 Cypher 证明可以在 TuGraph 中进行复杂图查询。
2. 再导入 Node2Vec 所需图数据。
3. 使用 `ceshi_node2vec.py` 完成 Node2Vec 的有偏随机游走采样。
4. 在采样序列基础上使用 Word2Vec 训练得到节点嵌入向量。
5. 最后将代码、报告和截图整理后提交到 GitHub，并把 GitHub 链接提交到学习通。
